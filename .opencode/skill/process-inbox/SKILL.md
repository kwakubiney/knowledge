## What I Do
I check the `inbox/` folder for queued context updates (generated by git hooks).
For each update, I evaluate if the `_context.md` needs changes.

## Workflow

1. **Scan Inbox** — Look for `inbox/update_*.md` files.
2. **Read Update** — Get the repo name, commit message, and diff stats.
3. **Load Target Context** — Read `vault/Hubtel/{Repo}/_context.md`.
4. **Evaluate** — Is this significant? (See `update-context` logic).
5. **Update Context** — If yes, append to Evolution Log.
6. **Archive** — Move the update file to `archive/inbox/`.

## Triggers
- "Check my inbox"
- "Any context updates?"
- "Process pending updates"

## Output
"Processed 3 updates.
- **payment-gateway:** Added 'Kafka Migration' to history.
- **consumer-app:** Ignored 'typo fix'.
- **settlements:** Updated architecture diagram."
